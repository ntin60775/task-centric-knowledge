## Цель

Формализовать полный bootstrap-контур для агента, который устанавливает knowledge-систему в новом проекте без интерактивных wizard. Агент должен иметь возможность одной командой или предсказуемой последовательностью команд развернуть knowledge-систему, открыть первую задачу и синхронизировать git-контекст. Текущий validated порядок из `references/adoption.md` требует ручного создания ветки и каталога задачи — эти шаги должны быть автоматизированы.

## Границы

### Входит

- Добавление команды `task-knowledge bootstrap --project-root /abs/project --profile generic` для полного bootstrap нового проекта.
- Bootstrap должен выполнять: install check → install apply → doctor-deps → создание первой task-ветки → создание первого task-каталога из шаблонов → register-if-missing.
- Обработка dirty tree: если дерево грязное из-за установки knowledge-файлов, bootstrap должен сначала сделать commit этих файлов или предложить стратегию.
- Поддержка профилей: `generic` и `1c`.
- Обновление `references/adoption.md` с новым упрощённым порядком.
- Добавление режима `--dry-run` для предварительного просмотра.

### Не входит

- Интерактивные wizard или prompt-ы.
- Миграция существующих систем хранения (это отдельный режим `migrate`).
- Создание задач с пользовательским содержимым (только из шаблонов).
- Изменение логики установки (install apply и doctor-deps остаются как есть).
