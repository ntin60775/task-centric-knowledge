# Операционные действия

`knowledge/operations/` предназначен для действий, которые помогают работе, но сами по себе не считаются задачами продукта.

## Что относится сюда

- сборка из исходников;
- создание и сопровождение служебных скриптов задач;
- выгрузка и загрузка артефактов;
- технические действия по среде;
- служебная диагностика и обслуживание.

## Что сюда не относится

Если действие меняет код, конфигурацию, схему данных, интерфейсы, прикладную логику, интеграцию, тесты или пользовательское поведение системы, это уже задача и ей место в `knowledge/tasks/`.

## Связь с задачами

Если операция выполняется в интересах конкретной задачи:

- в самой операции указывать связанный `ID` задачи;
- в `worklog.md` задачи давать ссылку на операцию, если это важно для сопровождения.

## Принцип простоты

Операции не должны разрастаться в отдельную систему планов.
Если операционное действие начинает жить собственной прикладной жизнью, его нужно поднять до полноценной задачи.
