# SDD: TASK-2026-0046

## Invariant set

- Обновление запускается только по явному запросу (нет авто-апдейта при global install).
- Список проектов хранится вне целевых проектов (в live-copy или user-config).
- Каждый проект обновляется атомарно: check → apply → verify. Если verify fails, изменения откатываются или фиксируется ошибка.
- Отчёт содержит абсолютные пути, статус ok/error и причину для каждого проекта.
- `--dry-run` не мутирует ни один проект.
- CLI/JSON surface новой команды стабилен.

## Допустимые связи

- `task_knowledge_cli.py` → новый модуль `mass_update.py` или внутри `install_skill_runtime`.
- `install_skill_runtime/environment.py` → `check()`, `install()`, `verify_project()` переиспользуются.
- Список проектов → JSON-файл в live-copy или `~/.config/task-knowledge/`.

## Недопустимые связи

- Нельзя хардкодить пути проектов в коде.
- Нельзя обновлять проекты без verify-project.
- Нельзя останавливаться при первой ошибке (batch-режим с отчётом).

## Новые зависимости

Нет новых runtime-зависимостей.
