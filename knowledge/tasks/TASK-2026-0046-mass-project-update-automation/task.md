## Цель

Автоматизировать массовое обновление knowledge-системы во всех установленных проектах. Вместо ручного запуска `task-knowledge install check/apply/verify-project` для каждого проекта, новая команда должна:

1. Обнаружить все проекты с установленной knowledge-системой.
2. Проверить состояние каждого проекта (check).
3. Обновить managed-файлы (apply --force).
4. Проверить результат (verify-project --force).
5. Собрать сводный отчёт.

## Границы

### Входит

- Новая подкоманда `task-knowledge install mass-update` (или `task-knowledge mass-update`).
- Механизм обнаружения проектов с установленной knowledge-системой.
- Пакетное выполнение check → apply → verify для каждого проекта.
- Поддержка фильтрации по профилю (`generic`, `1c`).
- Режим `--dry-run` для предпросмотра.
- Сводный отчёт в JSON и text форматах.
- Обработка ошибок: если один проект падает, остальные продолжаются.
- Интеграция с `scripts/install_global_skill.py` для определения source-root.

### Не входит

- Автоматический commit и push в проектах (это остаётся ручным шагом).
- Обнаружение проектов без knowledge-системы (только установленные).
- Параллельное обновление (последовательное для безопасности).
- Интерактивный выбор проектов.
- Изменение логики install apply/verify-project.
