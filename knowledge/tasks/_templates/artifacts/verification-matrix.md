# Матрица проверки по задаче TASK-2026-0001

## Когда использовать

Для любой задачи, где обязателен `sdd.md`, эта матрица по умолчанию тоже обязательна.
Её ведёт агент, а не пользователь.

## Паспорт

| Поле | Значение |
|------|----------|
| ID задачи | `TASK-2026-0001` |
| Связанный SDD | `../sdd.md` |
| Версия | `1` |
| Дата обновления | `YYYY-MM-DD` |

## 1. Канонические инварианты

Для code-related задач сюда по умолчанию должны попадать инварианты про:

- новые или запрещённые зависимости;
- допустимые и недопустимые связи между модулями, слоями или пакетами;
- источник истины, ownership по файлам и допустимые переходы состояний, если задача меняет contract или state-machine;
- критический функционал;
- основной сценарий;
- исходный наблюдаемый симптом, если задача поднята по логам, инциденту или деградации.

| Invariant ID | Описание | Источник истины | Где может сломаться |
|--------------|----------|-----------------|---------------------|
| `INV-01` | ... | ... | ... |
| `INV-02` | ... | ... | ... |
| `INV-03` | ... | ... | ... |

## 2. Матрица покрытия

| Invariant ID | Сценарий нарушения или переход | Автопроверка / команда | Статус покрытия | Примечание |
|--------------|--------------------------------|------------------------|-----------------|------------|
| `INV-01` | ... | `pytest ...` | `planned` | ... |
| `INV-02` | ... | `python3 -m unittest ...` | `covered` | ... |
| `INV-03` | ... | `git diff --check` | `covered` | ... |

Использовать фиксированный набор статусов покрытия:

- `planned` — проверка ещё не настроена или не прогнана;
- `covered` — покрытие доказано фактически выполненной проверкой;
- `manual-residual` — остался допустимый ручной остаток;
- `blocked` — инвариант сейчас нельзя доказать из-за внешнего блокера.

## 3. Остаточный риск и ручной остаток

- Если часть инвариантов осталась в `manual-residual`, перечислить их здесь и затем поднять в единый итоговый checklist общей задачи.
- Если остаточного риска нет, явно написать `нет`.

## 4. Правило завершения

- Задача не должна уходить в ревью, пока обязательные инварианты не связаны с проверками.
- Review не заменяет эту матрицу и не считается первым местом обнаружения базовых инвариантных ошибок.
