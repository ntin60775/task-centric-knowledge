## Цель

Сделать `task-centric-knowledge` навыком, который не только раскладывает task-артефакты, но и ведёт предсказуемый git-поток задачи по умолчанию.

Во второй итерации сделать правила выбора между текущей задачей, новой задачей и подзадачей настолько явными, чтобы система принимала решение автоматически по контексту, а при неоднозначности формулировала обязательный запрос с вариантами.

## Границы

### Входит

- skill `skills-global/task-centric-knowledge/`;
- локальные `knowledge/`-документы этого репозитория, которые должны соответствовать обновлённому skill-у;
- тесты и helper-скрипты, нужные для автоматизации git-потока задачи.
- правила task-routing для текущей задачи, новой задачи и подзадачи.

### Не входит

- push, PR и удалённые ревью-процессы;
- миграция всех существующих задач на новые ветки задним числом.

## Риски и зависимости

- Автоматические git-действия нельзя запускать поверх грязного дерева с несвязанными изменениями.
- Новые правила не должны противоречить базовым safety-ограничениям агента на destructive git-операции.
- Необходимо сохранить переносимость skill-а между репозиториями, несмотря на добавление helper-а.
